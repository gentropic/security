Verify it yourself — five minutes

Don't trust this page. Check it.

  1. Download lamina.html from the pinned release below.
  2. Hash it — shasum -a 256 lamina.html — it matches the published SHA-256 and the Zenodo DOI.
  3. Open it. DevTools → Network → load a file → zero requests.
  4. View Source — it is readable; here is the repository it is built from.
  5. Disconnect the network entirely. It still works.
  6. Export something — it writes only the one file you choose, through the operating-system Save dialog. Nothing silent; nowhere you did not pick.

The two questions a reviewer asks

Once you stop trusting file extensions as a proxy for capability, two coherent governance models remain — and a GCU artifact pre-answers both, by construction:

Govern the boundaryassume arbitrary compute, control reach → air-gapped, connect-src 'none', reach voluntarily zero — and demonstrable: pull the cable.
Govern provenanceidentity + integrity → hash-pinned, frozen, CC0-auditable — "is this the reviewed thing, unchanged."

Files for your tooling

Reviewer FAQ

Does it phone home?
No. connect-src 'none'; zero network requests; pull the cable and verify.
Does it need admin, or an installer?
No. It is a web document opened in your managed browser, under your CSP.
Can it read arbitrary files?
No. Only files you explicitly open; it cannot browse the disk.
Telemetry / analytics?
None. There is no network path for it to leave by.
How are updates and CVEs handled?
You pin a named release. You update when you choose, by pinning a new one. Zero runtime dependencies means a near-empty CVE surface.
Can we host our own copy?
Yes — that is the intended deployment. You hold a pinned copy; we operate nothing; uninstall = delete the file.
What if the author disappears?
Nothing breaks. It is public OSS (CC0 / MIT); your copy has no operator and no dependency on us being around.

What this is not

Not "certified safe." We don't sell a badge. Your security team owns the risk-acceptance decision; we give you what you need to make it.

Not installed-software cosplay. However it is packaged, a shortcut opens HTML in your browser under your CSP — packaging changes discovery, not what runs or where.

Not a support contract. It is public OSS; you run your own pinned copy. Authorship is disclosed, not hidden — where a deployment's author is also an employee, that is a conflict of interest to surface up front, and a reason the deploying organisation, not the author, owns the call.
